# 字段口径

把人格从标签改写成运行方式。所有字段都必须能回到证据。

## 来源字段

```text
记忆片段        可访问的历史表达、笔记、对话或用户粘贴内容
新回答          本轮缺口问题的回答
稳定模式        多个来源重复出现的行为倾向
证据缺口        当前无法判断但会影响结论的部分
```

不要写具体数量，除非你真的能数出来。不能数时写“少量”“多段”“当前对话”。

## 核心字段

```text
能量来源        用户从哪里恢复能量
能量消耗        什么环境会让用户快速耗竭
决策方式        用户做选择时优先保护什么
压力反应        用户受压后的自动动作
行动入口        用户最短启动路径
协作接口        别人如何给目标、边界、反馈
卡住位置        用户常把什么误认为前进
```

## 五维属性

分数只用粗略区间，不假装精确。

```text
自省 0-100      能否发现自己的卡点和循环
耐心 0-100      能否承受长周期、慢反馈
混沌 0-100      对变化、跳跃、即兴的亲和度
洞察 0-100      是否倾向抽象、归纳、看结构
锋利 0-100      对荒谬、低效、虚假叙事的敏感度
```

建议使用 5 分粒度，例如 35、60、75。不要输出 73 这种伪精确数字。

## 证据分级

```text
弱证据          单句自述、临时状态、玩笑话
中证据          多次出现的表达或一个具体场景
强证据          长期项目选择、反复行为、多个来源相互印证
反证            与候选类型或模式不一致的材料
```

反证必须出现。没有反证时写“当前材料不足，反证缺失”，并降低置信度。
