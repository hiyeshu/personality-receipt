# 类型 ASCII 字形库

这是没有上传图片时使用的稳定印章库。它不是装饰图库，而是人格运行模式的低分辨率图标。

## app.v1 输入契约

```text
type                白猫 / 狐狸 / 水母 / 兔子 / 其他扩展原型
rarity              自由 type 自带百分比，参考表只做校准锚点
glyphVariant        SOFT / RARE / GLITCH，可选
total               总判词，决定整体气质，例如 柔软但警觉
energy              决定身体重心、休息感、启动感
stress              决定眼神、缩起程度、防御姿态
decision            决定耳朵、触角、尾巴、边界方向
```

字段优先级：

```text
type     决定类型轮廓
rarity   决定票面稀有度百分比，由输入携带或参考表兜底
glyphVariant 决定稳定度和 glitch 程度
total    决定整体表情
energy   决定姿态松紧
stress   决定警觉、防御、抽离
decision 决定朝向、尖角、尾巴、手势
```

## 选择规则

```text
先看 data.type      明确写了类型原型时，直接选同名 glyph
再看运行模式       data.mbti / total / stress / energy 共同决定 fallback
最后用云团         证据不足、模式未定、字段缺失时用云团
```

不要临场自由画大图。每个印章控制在 3-5 行，宽度优先不超过 12 字符。ASCII 是印章，不是主菜。

## 网页端优先原型

这 4 个先服务 app fallback。没有上传图片时，优先从这里选。

白猫，柔软但警觉：

```text
 /\_/\
( o.o )
 > ^ <
```

狐狸，聪明、绕行、先试探再接近：

```text
 /\_/\
( o_o )
 /   \
  \_/
```

水母，敏感、漂浮、受环境电流影响：

```text
 .---.
( o o )
 ~|~|~
  | |
```

兔子，低攻击、快反应、容易受惊但恢复快：

```text
 /) /)
( . .)
(  > )
```

## 字形变体

字形变体只改变字形稳定感，不改变人格价值，也不改变稀有度百分比。

```text
SOFT      圆、松、留白更多，表情不攻击
RARE      轮廓更闭合，眼神更明确，结构更稳
GLITCH    允许一处错位、噪点或断裂，但不超过 1 行
```

变体例：

```text
SOFT 白猫
 /\_/\
( o.o )
 > ^ <

RARE 白猫
 /\_/\
( O.O )
/|_^_|\

GLITCH 白猫
 /\_/\_
( x.o )
 > # <
```
