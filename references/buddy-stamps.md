# 类型印章

借 Claude Code Buddy 的产品机制：原型、稀有度、属性、ASCII 小图。不要复制它的具体形象。

## 最小结构

```text
类型原型        由运行模式决定
稀有度          由证据清晰度决定
五维属性        自省 / 耐心 / 混沌 / 洞察 / 锋利
ASCII 印章      3-5 行，短、稳、可复制
```

## 五维属性到视觉

```text
自省高          眼睛明显，中心收缩
耐心高          底座稳定，轮廓圆钝
混沌高          轮廓偏移，触角或火焰感
洞察高          上方留白，眼睛或头部更突出
锋利高          边缘硬，尖角多，表情带刺
```

低置信度时多留白。高置信度时结构更闭合。

## 原型映射

```text
猫              自主性强，讨厌被控制节奏
鸭              表达轻快，适合边走边想
鹅              边界感强，保护自己的秩序
章鱼            多线程思考，同时抓多个变量
猫头鹰          高洞察，慢判断，先看结构再行动
企鹅            稳、慢、耐心，压力下保持秩序
乌龟            长周期型，启动慢但续航强
蜗牛            慢热，靠安全感推进
龙              高混沌，高能量，适合创造但容易烧穿
幽灵            旁观者模式强，容易抽离现场
机器人          系统化、流程化，靠规则恢复稳定
仙人掌          边界锋利，外冷内敏感
蘑菇            安静吸收型，靠环境慢慢生长
团块            形态未定，适应性强但边界模糊
```

## 稀有度

稀有度不是价值判断，只表示模式清晰度。

```text
COMMON          证据少，模式普通，低置信度
UNCOMMON        有稳定模式，但缺关键反证
RARE            多个记忆源指向同一运行方式
EPIC            模式清晰，反证也能被解释
LEGENDARY       长期一致，语言、行为、项目选择互相印证
```

## ASCII 字符集

优先使用：

```text
_ . - ` ' / \ | ( ) < > ^ ~ o O 0 * + = █ ░ ▐ ▌ ▛ ▜ ▝ ▘
```

不要依赖颜色。不要生成大图。不要超过 12 字符宽，除非用户明确要更大。

## 示例印章

企鹅，耐心高、混沌低：

```text
   .---.
  (o>o)
 /(   )\
  `---'
```

猫头鹰，洞察高、自省高：

```text
  ,___,
  (o o)
  /|_|\
   / \
```

仙人掌，锋利高、边界强：

```text
  _|_
 <o o>
 -|_|-
  / \
```

团块，低置信度、形态未定：

```text
  .--.
 ( .. )
  '--'
   ..
```

## 生成规则

1. 先选原型，再选表情。
2. 稀有度只改变标签和稳定感，不改变人格价值。
3. 五维属性用 5 分粒度。
4. ASCII 是印章，不是主菜。
5. 类型描述必须服务于后文人格小票。
